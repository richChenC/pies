# PIES 探头寿命统计分析软件

PIES 是一套面向探头检测业务场景的桌面分析工具，当前仓库维护的是基于 PyQt5 的主版本实现。软件用于从 Excel 文件或业务数据文件夹中提取探头使用记录，完成去重、统计、异常提示、历史累计、图表展示和结果导出，帮助使用者更稳定地完成探头寿命与检测数据分析。

## 1. 项目定位

这个项目主要解决三类问题：

1. 把分散在 Excel、业务数据文件夹、批量目录中的探头记录统一提取出来。
2. 把原始记录整理成可复核、可筛选、可统计、可导出的结构化结果。
3. 用统一口径输出探头寿命、使用次数、累计检测量、连续使用情况和图表结果，减少人工统计误差。

仓库当前重点面向实际生产环境中的日常分析需求，尤其适合以下场景：

- 分析单个 Excel 统计文件
- 批量处理本地业务数据文件夹
- 递归处理包含多个子目录的批量文件夹
- 只读方式读取共享目录或服务器映射盘中的数据
- 结合表格、图表和导出结果进行复核、留档和汇报

- 统计探头使用次数
- 统计累计检测管道数量

- 统计连续使用段
- 识别更换后再次使用情况

### 2.4 可视化展示

- 导出当前表格

- 导出当前图表

- 保存过程提供进度反馈

## 3. 统计口径

PIES 的价值不只是“把数据画出来”，更重要的是统一统计口径。当前版本的几个关键规则如下。

### 3.1 探头寿命口径

探头寿命以“累计检测管道数量”为主口径，不以单纯的累计使用时间作为寿命定义。

这意味着：

- 使用时间主要用于过程复核和辅助判断
- 寿命统计核心仍然落在检测量上
- 在业务汇报时，探头寿命更适合使用累计检测量表达

### 3.2 去重口径

去重不是只按探头编号判断，而是按完整记录字段判断。

只有当关键字段整体一致时，才视为重复记录。这样做的目的是：

- 避免误删真实业务记录
- 避免同一探头不同使用记录被错误合并
- 让去重结果能够追溯和解释

### 3.3 时间口径

开始时间和结束时间主要用于：

- 计算单条记录使用时长
- 识别异常时间记录
- 辅助判断连续使用
- 辅助判断更换后再次使用

时间并不是寿命的唯一依据，也不是探头价值判断的主指标。

### 3.4 连续使用识别

连续使用段并不是只靠“时间差小于多少分钟”简单判断，而是综合以下因素：

- 时间间隔
- `SG_ID`
- 数据组编号
- 中间是否有其他探头占用

因此，软件对“连续使用”和“更换后再次使用”的识别更接近实际业务场景，而不是纯粹的时间排序算法。

### 3.5 特殊标记处理

像 `888`、`999` 这类非真实检测标记，不应被当成真实检测量参与核心寿命统计。

## 4. 主要业务流程

### 4.1 Excel 分析流程

1. 启动软件
2. 选择“分析 Excel”
3. 选择待分析文件
4. 软件提取记录并执行清洗、去重、统计
5. 在表格中查看明细和汇总
6. 在图表区查看可视化结果
7. 根据需要导出表格、图表或执行一键保存

### 4.2 文件夹批量分析流程

1. 选择“处理文件夹”
2. 选择本地目录或共享目录
3. 软件自动扫描有效数据目录
4. 对每个最小有效文件夹执行提取与统计
5. 汇总结果展示到表格与图表中
6. 根据需要保存结果

### 4.3 历史累计流程

1. 导入或处理当前批次数据
2. 检查异常提示和去重结果
3. 将当前批次纳入历史累计
4. 切换到历史累计视图查看长期统计结果

## 5. 界面功能概览

### 5.1 表格区

表格区用于查看原始记录、统计结果和筛选后的内容，支持：

- 表头下拉筛选
- 多选筛选
- 手动输入筛选
- 表格与图表联动刷新

常见表格字段包括：

- 大修/批次
- 蒸汽发生器编号
- 数据组
- 操作员
- 探头类型
- 探头编号
- 探头型号
- 管道数量
- 累计管道数量
- 开始时间
- 结束时间

### 5.2 图表区

图表区用于按不同维度查看统计结果，支持：

- 按探头编号查看
- 按探头类型查看
- 按探头型号查看
- 按批次查看
- 在不同图表风格间切换

当前版本支持的展示预设包括：

- 均衡显示
- 清爽少标注
- 重点突出
- 汇报展示

这些预设只改变显示方式，不改变统计结果。

### 5.3 辅助窗口

辅助窗口主要用于查看：

- 软件说明
- 数据安全提示
- 异常提示
- 去重详情
- 历史详情

这些窗口支持重复打开、刷新内容和调整大小，便于复核时并排查看。

## 6. 数据安全原则

PIES 在设计上强调“源数据只读”。

尤其是网络共享目录和服务器映射盘，软件应遵循以下原则：

- 不修改源文件
- 不删除源文件
- 不重命名源文件
- 不向源目录写缓存
- 不向源目录写历史数据
- 不向源目录写导出结果

导出结果、缓存、历史数据和本地设置应写入本地工作目录，例如 `SaveDate/` 或本机其他本地目录。

## 8. 运行环境

建议环境：

- Windows 桌面环境
- Python 3.10 及以上

主要依赖：

- PyQt5
- pandas
- openpyxl
- matplotlib
- numpy

安装依赖：

```bash
pip install -r requirements-pyqt.txt
```

启动方式：

```bash
python run.py
```

或者：

```bash
python run_pyqt.py
```

程序运行后会自动创建日志目录并记录运行日志，便于排查问题。

## 9. 常用导出结果

当前版本支持输出的结果主要包括：

- 当前明细表
- 当前统计表
- 当前图表
- 总图
- 子图
- 一键保存结果包

图表导出会尽量基于原始绘图数据重新生成，而不是简单裁切总图，以减少空白图、重复图和标题错误的问题。

## 10. 异常与复核重点

软件重点识别和提示以下问题：

- 管道数量为 0
- 开始时间为空
- 结束时间为空
- 开始和结束时间同时为空
- 时长小于等于 0
- 时间顺序异常
- 数据组缺少关键标记
- 特殊占位标记混入真实统计

建议在正式导出前，先查看：

- 异常提示
- 去重详情
- 历史累计结果
- 图表与表格是否一致

## 11. 上线前建议检查

1. 启动后默认图表是否正常显示
2. 高频切换筛选后表格和图表是否同步刷新
3. 一键保存和批量导出是否有明确进度反馈
4. 辅助窗口是否可以重复打开并正常刷新
5. 网络目录导入是否始终保持只读
6. 导出的图表标题、命名、坐标轴和数值标注是否一致
