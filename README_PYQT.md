# PIES PyQt 版本说明

## 1. 版本定位

本版本是 PIES 的 PyQt 桌面实现，也是当前仓库中的主要运行版本。

程序入口：

- `run.py`
- `run_pyqt.py`

依赖文件：

- `requirements-pyqt.txt`

核心目录：

- `pies_pyqt/core/`
- `pies_pyqt/ui/`
- `pies_pyqt/workers/`
- `pies_pyqt/resources/`

## 2. 设计目标

PyQt 版本的主要目标如下：

- 提供完整的桌面图形界面
- 让导入、分析、查看、导出流程可视化
- 适配较大批量的数据处理场景
- 尽量减少界面卡顿、无响应和重复计算
- 支持本地使用和共享目录只读导入

## 3. 已覆盖能力

### 3.1 数据导入

- 导入 Excel 数据并直接分析
- 读取本地目录中的业务数据文件
- 支持递归处理包含子目录的数据目录
- 支持从共享目录读取数据

### 3.2 数据分析

- 探头记录提取
- 记录去重
- 异常记录提示
- 当前批次统计
- 历史累计统计
- 探头编号维度统计
- 探头类型维度统计
- 探头型号维度统计

### 3.3 展示与导出

- 表格结果展示
- 图表展示
- 单图导出
- 批量导出
- 一键保存图表与表格

### 3.4 交互能力

- 表头筛选
- 手动条件筛选
- 当前批次与历史累计切换
- 警告查看
- 去重详情查看
- 软件说明查看

## 4. 统计规则补充

### 4.1 去重规则

去重按完整记录字段判断，不是只按探头编号简单去重。

### 4.2 时间规则

时间主要用于：

- 计算单条记录使用时长
- 识别连续使用段
- 辅助判断更换后再次使用
- 生成异常提示

### 4.3 连续使用识别

连续使用段判断不仅看时间间隔，还会结合：

- `SG_ID`
- 数据组编号
- 中间是否出现其他探头占用

## 5. 数据安全原则

建议遵循以下原则：

- 源数据目录只读
- 不修改原始文件
- 不在源目录写缓存
- 不在源目录写历史数据
- 导出结果保存到本地工作目录

## 6. 常用检查脚本

仓库内可直接运行：

```bash
python tools/regression_smoke.py
python tools/benchmark_large_flow.py
python tools/stress_chart_cycles.py
python tools/real_data_end_to_end_check.py
```
