# PIES PyQt 版本说明

本文档用于补充说明当前仓库中的 PyQt 版本实现，重点描述版本定位、设计目标、模块划分、性能与交互策略，以及推荐验收方式。

## 1. 版本定位

当前仓库中的主运行版本为 PyQt5 桌面版，入口如下：

- `run.py`
- `run_pyqt.py`

依赖文件：

- `requirements-pyqt.txt`

PyQt 版本承担的是实际业务使用中的主分析流程，目标不是做最小化演示，而是提供可以长期维护、可复核、可导出的桌面分析工具。

## 2. 设计目标

PyQt 版本的核心设计目标包括：

- 提供完整、稳定、直接的桌面操作流程
- 支持 Excel、本地目录、批量目录和共享目录导入
- 让表格、图表、异常提示、去重详情和导出流程保持一致
- 尽量降低界面卡顿、空白图、重复图和状态错乱问题
- 让复杂数据处理尽量放到后台任务中执行

## 5. 统计与识别逻辑补充

### 5.1 去重逻辑

去重按完整记录字段匹配，而不是只按探头编号匹配。

这样做的目的是：

- 保留真实业务差异
- 避免误去重
- 提升复核可信度

### 5.2 连续使用识别

连续使用识别不仅依赖时间间隔，还会结合：

- `SG_ID`
- 数据组编号
- 中间是否存在其他探头使用记录

因此，这里的“连续使用段”更偏向业务语义，而不是纯粹时间切片。

### 5.3 时间与寿命关系

时间数据主要用于过程复核和辅助判断，不直接等同于寿命定义。

寿命核心仍然以累计检测量为主。

## 6. 数据安全原则

对源数据目录，尤其是网络共享目录，应始终保持只读原则：

- 不修改源文件
- 不删除源文件
- 不重命名源文件
- 不在源目录写缓存
- 不在源目录写导出结果
- 不在源目录写历史累计数据

推荐做法：

- 从共享目录读取数据
- 把导出结果写到本地目录
- 把历史累计写到本地目录

## 7. 性能与稳定性关注点

当前 PyQt 版本重点处理过的典型问题包括：

- 启动后图表首次显示异常
- 高频切换筛选后的空表或空图
- 图表重绘过多导致界面卡顿
- 子图导出重复、空白或标题错误
- 长操作缺少进度反馈
- 辅助窗口状态不同步

围绕这些问题，当前实现更强调：

- 后台处理
- 缓存复用
- 轻量刷新
- 明确进度
- 尽量减少重复计算

## 8. 推荐回归检查

建议至少检查以下内容：

1. 启动软件后默认图表是否正常显示
2. 高频切换筛选条件后，表格和图表是否同步刷新
3. 导出当前图表、总图和子图时，文件内容与命名是否正确
4. 一键保存过程中是否存在无响应或重复触发
5. 打开说明、警告、去重、历史等窗口后，是否能重复打开并刷新内容
6. 使用网络共享目录导入时，是否全程保持只读
