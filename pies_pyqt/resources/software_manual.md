# PIES 软件使用说明

## 1. 软件定位

PIES 用于从 Excel 或 SUM/ECT 文件夹中提取探头使用记录，完成去重、统计、历史累计、异常提示、图表可视化与表格导出。

软件原则：

1. 只在本地处理副本数据，不回写原始文件。
2. 当前批次与历史累计分开保存，可切换查看。
3. 所有图表和统计都基于程序内统一的分析结果生成，避免口径不一致。

## 2. 主界面功能

### 2.1 文件菜单

- `处理文件夹`
  - 解析 SUM/ECT 文件夹，提取探头记录。
- `分析Excel`
  - 导入现成 Excel 表格并分析。
- `一键保存`
  - 批量导出当前统计表、原始记录表和图表图片。
- `退出`
  - 关闭软件。

### 2.2 视图菜单

- `信息表格`
  - 查看明细表和统计表。
- `数据可视化`
  - 查看折线图、柱状图、饼图等。

### 2.3 历史菜单

- `开启/关闭累计历史`
  - 决定当前分析结果是否自动并入本地历史仓。
- `切换到当前批次`
  - 只看最近一次导入分析结果。
- `切换到历史累计`
  - 看本机累计后的历史总数据。
- `历史详情`
  - 查看历史总量、导入次数、累计去重等信息。
- `清空数据`
  - 清空当前批次和本地历史累计数据。

### 2.4 导出与提醒菜单

- `导出表格`
  - 导出当前表格或当前统计结果。
- `导出错误记录`
  - 导出无法正常解析或存在严重问题的记录。
- `查看警告`
  - 查看时间异常、管道数异常、空值异常等警告。
- `去重信息`
  - 查看原始记录数、去重后数量、移除重复数，以及被移除的重复记录详情。

### 2.5 帮助菜单

- `软件使用说明`
  - 打开本说明文档。
- `数据安全提示`
  - 查看本地处理和副本分析提醒。

## 3. 常用操作流程

### 3.1 Excel 分析流程

1. 点击 `分析Excel`。
2. 选择 Excel 文件。
3. 软件提取记录、执行去重、生成统计。
4. 在 `信息表格` 查看原始记录和探头汇总。
5. 在 `数据可视化` 查看图表。
6. 需要保存时，使用 `导出表格` 或 `一键保存`。

### 3.2 文件夹分析流程

1. 点击 `处理文件夹`。
2. 选择包含 SUM/ECT 的目录。
3. 软件解析每个数据组，生成记录表。
4. 自动执行去重与统计。
5. 在图表和表格中查看结果。

### 3.3 历史累计流程

1. 保持 `历史记录：开`。
2. 每次分析完成后，当前批次数据会按唯一记录合并进本地历史仓。
3. 切换到 `历史累计` 可查看累计后的有效记录和统计结果。

## 6. 关键统计原理

### 6.1 去重逻辑

只有当两条记录以下字段全部一致时，才判定为重复记录：

- 大修
- SG 编号
- 数据组
- 操作员
- 探头类型
- 探头编号
- 探头型号
- 管道数量
- 开始时间
- 结束时间

也就是说，去重是“完整行匹配”，不是只按探头编号去重。

### 6.2 总使用时间

每条有效记录的使用时间 = `结束时间 - 开始时间`。

有效记录通常要求：

- 开始时间有效
- 结束时间有效
- 时长大于 0
- 管道数不为 0
- 时间不是异常占位值（例如 1900 年）

探头总使用时间 = 该探头所有有效记录使用时间之和。

### 6.3 最长连续使用时间

软件会先把同一探头的记录按时间排序，再拆分成多个“使用段”。

相邻两条记录通常会被视为同一连续使用段，条件是：

1. 时间间隔不大于 30 分钟。
2. 如果处于同一 `SG_ID`，则数据组编号连续性合理。
   - 单推场景常见递增 `+1`
   - 双推交替场景常见递增 `+2`
3. 如果数据组跳跃较大，但中间没有检测到其他探头插入使用，也仍可能视为同一连续段。

每个使用段的时长 = 该段最早开始时间到最晚结束时间。

`最长连续使用时间` = 所有使用段中最长的一段。

### 6.4 更换后再次使用

如果同一探头被拆成了多个使用段，说明它不是一直连续用完的，中间发生了停用后再次投入。

程序判断依据：

1. 先按连续使用规则拆段。
2. 在两个相邻使用段之间，检查同一 `SG_ID` 下是否有其他探头被使用。
3. 如果中间确实换成了别的探头，再换回当前探头，就判定为“更换后再次使用”。

界面中常见两个相关口径：

- `使用段数`
  - 等于该探头被拆分出的使用段数量。
- `再次使用次数`
  - 本质上反映该探头存在再次投入的情况；使用段数大于 1 时，说明发生过再次使用。

### 6.5 检测速度

检测速度 = 累计检测管道数量 / 总使用小时数。

### 6.6 生产批次平均寿命

根据探头编号规则解析生产年月，再按批次分组统计平均寿命。

说明：

- MRPC 6 位/7 位编号按前三位识别生产年月。
- `ZRPF-FH/2C`、`ZRPC-FH/2C` 作为老型号提示显示，但不参与寿命规则统计。

## 7. 文件夹解析规则

### 7.1 时间来源

处理文件夹时：

- `Start Time` 和 `End Time` 取对应数据组开组/结束标记文件的时间。
- 时间精确到秒。

### 7.2 数据组异常处理

如果一个数据组缺少关键标记文件：

- 软件会给出警告或错误提示。
- 对于无法可靠恢复的数据组，会跳过该组，避免生成错误统计。

### 7.3 管道数规则

`Tube Number` 只统计真实管子数量，不将 `999`、`888` 等非真实检测标记文件计入真实管道数。

### 8.1 导出内容

- 导出当前明细表
- 导出当前汇总表
- 导出去重表格
- 导出错误记录
- 一键导出图表和表格

## 9. 异常与警告说明

软件会重点识别以下问题：

- 管道数量为 0
- 开始时间为空
- 结束时间为空
- 开始时间与结束时间都为空
- 开始时间和结束时间相同
- 使用时长小于等于 0
- 数据组缺失关键时间标记
- SUM/ECT 文件损坏或格式异常

原则：

1. 能提示的尽量提示。
2. 会影响统计可靠性的记录尽量不纳入核心统计。
3. 保留警告信息，便于复核原始数据质量。

## 10. 历史累计说明

历史累计数据保存在本机 `probe_history.json` 中。

口径说明：

- 历史累计的“去重后数量”是当前历史仓中实际保留的有效记录数。
- 历史累计的“移除重复数”与当前可展示的去重明细保持一致。
- 历史累计用于跨批次分析，不影响原始 Excel 或原始文件夹数据。

## 11. 推荐使用习惯

1. 先在本地副本上分析，不直接处理服务器原始目录。
2. 每次导入后先看 `查看警告`。
3. 大批量分析时优先使用文件夹副本，避免网络盘卡顿。
4. 发现异常统计时，先核对去重信息、警告信息和原始明细表。
5. 定期导出关键结果，便于留档。

## 12. 当前版本已经特别处理的问题

- 历史累计去重统计与去重明细条数统一口径，避免“卡片数字与表格行数不一致”。
- 切换当前批次/历史累计时复用已有统计结果，减少重复重算。
- 连续使用识别时按 `SG_ID` 预筛候选记录，降低大批量分析时的扫描开销。
- 去重窗口按钮明确为 `导出去重表格`，避免和普通表格导出混淆。
